Zero Touch Provisioning (ZTP) is a configuration method for Arista Network switches that does not require user intervention. It works by downloading a startup configuration file (startup-config) or a boot script from a location specified by a DHCP server. The switch enters ZTP mode when it boots if flash memory does not contain a startup-config. It remains in ZTP mode until a user cancels ZTP mode, or until the switch retrieves a startup-config or a boot script.

To cancel ZTP mode, you need to log into the switch using the admin password, then enter the 'zerotouch cancel' command. The switch will then boot immediately without installing a startup-config file. To avoid entering ZTP mode on subsequent reboots, a startup-config file needs to be created.
